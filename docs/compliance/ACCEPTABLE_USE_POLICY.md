# Acceptable Use Policy

**Evenfall Advantage LLC — Overwatch Platform**
**Effective Date:** April 6, 2026
**Owner:** James Ferguson, CTO

---

## 1. Purpose

This policy defines acceptable and prohibited uses of Evenfall Advantage LLC information systems, including the Overwatch platform, development tools, and communication channels.

## 2. Scope

Applies to all employees, contractors, and authorized third parties who access company systems.

## 3. Acceptable Use

### 3.1 General
- Use company systems for authorized business purposes
- Protect login credentials — never share passwords or API keys
- Lock workstations when unattended (5-minute auto-lock required)
- Report suspected security incidents immediately to CTO

### 3.2 Devices
- Enable full-disk encryption on all devices used for work (FileVault, BitLocker)
- Keep operating systems and software up to date
- Use antivirus/endpoint protection software
- Enable screen lock with password/biometric
- Personal devices used for work must meet the same security requirements

### 3.3 Authentication
- Use unique, strong passwords (minimum 12 characters) for all work accounts
- Enable multi-factor authentication (MFA) on: GitHub, Supabase Dashboard, Stripe Dashboard
- Never reuse passwords across services
- Use a password manager

### 3.4 Development
- Never commit secrets, API keys, or credentials to source code
- Follow the change management process (PRs with review)
- Test changes before deploying to production
- Report security vulnerabilities through the responsible disclosure process

### 3.5 Data Handling
- Handle customer data according to its classification (see Information Security Policy)
- Do not store Restricted or Confidential data on personal devices without encryption
- Do not transmit Restricted data over unencrypted channels
- Delete local copies of customer data after use

## 4. Prohibited Use

- Sharing credentials or authentication tokens with unauthorized parties
- Accessing systems or data beyond your authorized role
- Disabling or circumventing security controls (firewalls, RLS, MFA)
- Installing unauthorized software on production systems
- Using company systems for illegal activities
- Storing or transmitting customer data through personal email or messaging
- Connecting to production databases from unsecured networks without VPN
- Ignoring security alerts or vulnerability notifications

## 5. Enforcement

Violations of this policy may result in:
- Verbal or written warning
- Temporary suspension of access
- Termination of employment/contract
- Legal action (if criminal activity is involved)

## 6. Review

This policy is reviewed annually and updated as needed.
