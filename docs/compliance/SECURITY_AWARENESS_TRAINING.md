# Security Awareness Training Program

**Evenfall Advantage LLC — Overwatch Platform**
**Effective Date:** April 6, 2026
**Owner:** James Ferguson, CTO

---

## 1. Purpose

This program ensures all personnel understand their security responsibilities and can identify and respond to common threats.

## 2. Scope

Required for all employees, contractors, and authorized third parties who access Overwatch or handle customer data.

## 3. Training Schedule

| Training | Frequency | Audience | Duration |
|----------|-----------|----------|----------|
| New hire onboarding security briefing | At hire | All new personnel | 30 min |
| Annual security refresher | Annually (Q1) | All personnel | 45 min |
| Incident response drill | Annually (Q2) | Technical team | 1 hour |
| Role-specific training (admin/owner) | At role assignment | Admins/owners | 30 min |

## 4. Core Training Topics

### 4.1 New Hire Onboarding (30 min)
- [ ] Overview of Overwatch platform and data it handles
- [ ] Data classification levels (Restricted, Confidential, Internal, Public)
- [ ] Acceptable Use Policy review and acknowledgment
- [ ] Code of Conduct review and acknowledgment
- [ ] Password requirements (12+ characters, no reuse, password manager)
- [ ] Multi-factor authentication setup (GitHub, Supabase, Stripe)
- [ ] Device security requirements (encryption, screen lock, updates)
- [ ] How to report security incidents (email CTO, Briefing post)
- [ ] RBAC overview — understanding your role and access level

### 4.2 Annual Refresher (45 min)
- [ ] Review of security incidents from the past year (anonymized)
- [ ] Updated threat landscape (phishing, social engineering, supply chain)
- [ ] Phishing identification exercise (recognize suspicious emails/links)
- [ ] Password hygiene check (verify no reused passwords, MFA active)
- [ ] Review of policy updates (any changes since last training)
- [ ] Data handling best practices refresher
- [ ] Q&A session

### 4.3 Phishing Awareness
- How to identify phishing emails (suspicious sender, urgency, mismatched URLs)
- What to do if you receive a phishing email (don't click, report to CTO)
- What to do if you clicked a phishing link (change password immediately, report)
- Examples of real-world phishing attacks targeting security companies

### 4.4 Admin/Owner Role Training (30 min)
- Elevated access responsibilities
- How RLS and company isolation works
- Edge Function security (JWT, admin verification)
- How to review audit logs and error logs
- Personnel management security (applicant data, hire process)
- How to use the access review checklist

## 5. Training Delivery

| Method | Use Case |
|--------|----------|
| In-person / video call | New hire onboarding, annual refresher |
| Self-paced document review | Policy acknowledgments |
| Tabletop exercise | Incident response drill |
| Overwatch Briefing | Security tips and reminders (monthly) |

## 6. Tracking and Records

- Training completion recorded in personnel records
- Policy acknowledgments signed and dated
- Annual refresher attendance tracked
- Incomplete training escalated to management within 30 days

## 7. Metrics

| Metric | Target |
|--------|--------|
| New hire training completion | 100% within first week |
| Annual refresher completion | 100% by end of Q1 |
| Phishing exercise pass rate | > 80% |
| Policy acknowledgment rate | 100% |

## 8. Review

This program is reviewed annually and updated based on emerging threats and incident lessons learned.
